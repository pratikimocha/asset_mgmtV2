Microsoft 365 Graph API Setup Guide

Office 365 / Outlook via Azure App Registration (application permissions — no user sign-in needed).

  1. In Azure Portal → Azure Active Directory → App registrations, click New registration.
  2. Under API permissions → Add → Microsoft Graph → Application permissions, add Mail.Send and Mail.ReadWrite, then Grant admin consent.
  3. Under Certificates & secrets → New client secret, copy the Value (not the ID).
  4. Add these to your .env:
MAIL_PROVIDER=graph
MAIL_TENANT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MAIL_CLIENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MAIL_CLIENT_SECRET=your-secret-value-here
MAIL_MAILBOX=helpdesk@yourcompany.com
MAIL_FROM=iMocha Helpdesk <helpdesk@yourcompany.com>
MAIL_AUTO_REPLY=1
MAIL_POLL_INTERVAL=60

Restart the server after saving. The Graph inbox poller runs automatically every MAIL_POLL_INTERVAL seconds. MAIL_MAILBOX must be the shared mailbox or user mailbox the app has been granted access to.

Gmail / IMAP Setup Guide

  1. Enable 2-Step Verification on your Google account.
  2. Go to Google Account → Security → App Passwords and create an app password for "Mail".
  3. Add these to your .env file:
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=support@yourcompany.com
MAIL_PASSWORD=xxxx-xxxx-xxxx-xxxx
MAIL_FROM=iMocha Helpdesk <support@yourcompany.com>
MAIL_USE_TLS=1

MAIL_IMAP_SERVER=imap.gmail.com
MAIL_IMAP_PORT=993
MAIL_AUTO_REPLY=1
MAIL_POLL_INTERVAL=60

Restart the server after saving. The IMAP poller runs automatically every MAIL_POLL_INTERVAL seconds.

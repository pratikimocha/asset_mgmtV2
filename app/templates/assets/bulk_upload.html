CSV Format

Your CSV file must have these column headers (first row):

serial_number,model,manufacturer,status,assigned_to,asset_tag,category,purchase_date,warranty_expiry,cost,vendor,location,department,assigned_date,issue_text,issue_severity,lifecycle_event,lifecycle_event_type,lifecycle_event_date
ColumnRequiredNotes
serial_numberAlwaysUnique identifier per asset
modelAlwaysDevice model name (e.g. MacBook Pro 14)
manufacturerAlwaysBrand (e.g. Apple, Dell, HP)
statusAlwaysinstock / deployed / repair / sold / retired / ordered / received
assigned_toIf deployedEmployee name — required when status is deployed
asset_tag—Internal tag (e.g. IT-0042)
category—e.g. Laptop, Mobile, Monitor
purchase_date—YYYY-MM-DD
warranty_expiry—YYYY-MM-DD
cost—Number (e.g. 75000)
vendor—Supplier name
location—Office / branch
department—Team name
assigned_date—YYYY-MM-DD (defaults to today)
issue_text—Issue description (auto-creates an open issue)
issue_severity—low / medium / high (defaults to medium)
lifecycle_event—Text note shown in the asset's Lifecycle tab (e.g. "Repaired screen", "Assigned to IT dept")
lifecycle_event_type—Event type label (e.g. repair, assignment, note, purchase). Defaults to note
lifecycle_event_date—YYYY-MM-DD — when the event happened. Defaults to purchase_date or today
